# ExampleCo Vendor Risk Guide

This is synthetic vendor-risk reference material for the Agent2 procurement
compliance officer example.

## Vendor risk levels

Low-risk vendors are approved vendors with routine categories and no known
privacy, sanctions, security, or payment concerns.

Medium-risk vendors are usable but require conditions such as security review,
data protection review, updated tax forms, or finance review.

High-risk vendors require legal, finance, or security escalation before approval.

Blocked vendors must not be approved. A blocked vendor is a rejection case unless
an explicit exception procedure is documented in a separate policy package.

## Known example vendors

Acme Hardware is a low-risk preferred vendor for laptops, monitors, docking
stations, and standard peripherals.

CollabCloud is approved with conditions. It is a medium-risk cloud software
vendor. It requires security review and a data processing agreement when it
handles source code, employee data, customer data, or shared files.

Northstar Intermediaries is blocked. The vendor has a sanctions-screening match
in this synthetic example. Requests involving this vendor must be rejected.

## New vendor onboarding

New vendors are not automatically rejected. The officer should ask for missing
onboarding information when needed:

- legal vendor name
- tax form or registration number
- payment details
- security review status for software or data processors
- contract owner

## Risk memory

The officer may keep concise vendor memory after each case. Memory should record
stable operating facts such as common category, required review, relationship
owner, and previous clarification patterns. Memory must not replace policy books.
